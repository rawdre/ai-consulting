# Mission Control Lanes

Each lane exists to support Comeketo and Close-first execution. Done means the lane produced a useful operational output, not just internal activity.

## CRM / Follow-Up Lane

- Owning agent(s): Rawbot 🔥, Ogilvy
- Mission: turn lead context into the next best follow-up move
- Inputs: Close guardrail snapshot, lead stage, unanswered messages, follow-up debt
- Outputs: reply drafts, follow-up recommendations, visible lead priorities
- Done looks like: Andre can see who needs contact now and what message logic should be used

## Call / Tasting Conversion Lane

- Owning agent(s): Cialdini, Rawbot 🔥
- Mission: move qualified leads toward calls, tastings, and booked events
- Inputs: opportunity stage, lead heat, tasting status, stalled deals
- Outputs: tasting invites, confirmation pushes, conversion pressure guidance
- Done looks like: Mission Control highlights the highest-leverage conversion moves instead of generic follow-up

## Automation / Intake Lane

- Owning agent(s): Turing, Linus
- Mission: keep lead intake and resurrection paths clean and reliable
- Inputs: forms, inboxes, Expo/event leads, archive candidates, intake rules
- Outputs: canonical intake path, automation tasks, system handoff fixes
- Done looks like: new leads land in the right operational path without manual confusion

## Dashboard / UI Lane

- Owning agent(s): Ada
- Mission: make Mission Control scannable, premium, and operator-useful
- Inputs: current dashboard data, squad accountability payload, operator needs
- Outputs: dashboard sections, readable layout, compact accountability surface
- Done looks like: Andre can glance at the page and know what matters without hunting

## Engineering / Debug Lane

- Owning agent(s): Linus
- Mission: keep the repo, data pipeline, and Close guardrail reliable
- Inputs: code defects, stale data, broken scripts, stack inconsistencies
- Outputs: fixes, refactors, stable build paths, cleaner data shape
- Done looks like: the dashboard refreshes cleanly and the codebase stays maintainable

## Prioritization / ROI Lane

- Owning agent(s): Buffett
- Mission: decide what deserves time, automation, or spend
- Inputs: task backlog, conversion leverage, maintenance cost, operating friction
- Outputs: priority judgment, stop-doing calls, automation worthiness decisions
- Done looks like: low-value work is filtered out and high-value work gets attention first

## Voice / Media Lane

- Owning agent(s): Spielberg
- Mission: add story or media only when it helps the sale or brand trust
- Inputs: campaign needs, presentation needs, video or voice opportunities
- Outputs: media concepts, story framing, voice-led assets
- Done looks like: a media output directly supports a live sales or brand objective

## Doctrine / Continuity Lane

- Owning agent(s): LenoRawbot
- Mission: preserve rules, continuity, and operating clarity
- Inputs: doctrine updates, system constraints, coordination failures, review notes
- Outputs: clarified rules, memory docs, status structure, accountability hygiene
- Done looks like: the team can explain the system simply and operate it without drift

## Overall Sequencing Lane

- Owning agent(s): Rawbot 🔥, Buffett
- Mission: decide what happens next across lanes
- Inputs: current priorities, blocks, leverage, deadlines, funnel pressure
- Outputs: next action order, active focus, execution sequence
- Done looks like: Mission Control answers what Andre should do next and why

## 7. Current Risks

### Risk 1 — Multiple environments
Some bots may report paths/files from different runtime environments.
This can create confusion between:
- André's local machine
- another bot's local workspace
- live GitHub repo state

**Rule:** environment differences must be made explicit in reports.

### Risk 2 — Parallel structures
Without enforcement, bots may create:
- duplicate folder systems
- duplicated files
- conflicting “official” sources

**Rule:** canonical structure is controlled by Rawbot.

### Risk 3 — Publish confusion
Site work can be confused between:
- editable source
- publish repo
- root workspace clutter

**Rule:** publish flow must follow documented sources and repos.
